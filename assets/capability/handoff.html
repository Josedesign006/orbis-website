08 Token-adoption gaps

The most actionable cleanup list — raw values that bypass the system, in priority order.

  • No spacing / radius / shadow tokens. theme.css tokenises colour, type and motion only — every box-model value is a raw inline px. Biggest systemisation opportunity.
  • ReaderScreen THEMES hardcodes 15 hex values, including a paper theme that duplicates --paper / --ink / etc. instead of referencing the token layer.
  • #E03A28 appears literally in 3 spots (range accentColor, theme-button border, TBtn active) — should be var(--red).
  • --red-deep and --line are defined but never used — wire them up or remove.
  • #1b1814 backdrop is a raw literal in index.css.
  • Tailwind CDN is loaded but unused — drop it, or adopt it; carrying both is dead weight.
  • Hit targets inconsistent — Library icon buttons 40×40 (below 44px); Detail uses 44×44. Normalise to ≥44.
  • No :hover states anywhere — acceptable for touch, but this is a react/web handoff, so add hover affordances for pointer devices.
